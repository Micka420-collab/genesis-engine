```python
GDAL_DISABLE_READDIR_ON_OPEN = EMPTY_DIR
```

### Bonus — Bug pré-existant sur dashboard `/api/state` corrigé ✅
**Modifiés :** `runtime/engine/annalist.py`, `runtime/engine/dashboard.py`

Symptôme : `/api/state`, `/api/agents`, `/api/metrics` retournaient HTTP 200 avec body vide.
Cause : `sim.snapshot()` appelait `self.annalist.wall_clock_s()` qui n'existait pas → AttributeError dans `json.dumps()` après l'envoi du status code → response tronquée.
Fix :
1. Ajout de `Annalist.wall_clock_s()` (calcule `time.monotonic() - self._start_ts`).
2. Ajout d'un `_json_default()` numpy-aware dans `dashboard.py` et wrap `_json()` avec try/except → 500 + erreur JSON parseable si serialization échoue.

Vérifié : tous les `/api/*` retournent maintenant 200 + JSON valide (`/api/state` 239 bytes, `/api/agents` 2029 bytes, `/api/metrics` 2542 bytes).

### P5 (L2) — Module `sim_lift.py` créé + smoke test PASSED ✅
**Nouveau module :** `runtime/engine/sim_lift.py` (~260 lignes)
**Test :** `runtime/scripts/p5_lift_smoke.py` + `journals/p5_lift_smoke.jsonl`

**Mécaniques implémentées :**
- Succession végétale en 5 états (Markov) : `PRAIRIE → GARRIGUE → BOIS_JEUNE → FORET_MATURE → FORET_VIEILLE`
- Constantes de temps réalistes : 6 ans (PRAIRIE→GARRIGUE), 12 ans (→BOIS_JEUNE), 35 ans (→FORET_MATURE), 90 ans (→FORET_VIEILLE)
- Cap par biome : forêts montent jusqu'à FORET_VIEILLE, shrub-biomes capés à GARRIGUE, désert/océan à PRAIRIE
- Initialisation hétérogène : ~10% des cellules forestières démarrent en FORET_VIEILLE, ~5% en BOIS_JEUNE → texture immédiate
- `chunk.wood` est blendé vers `biome_base × veg_multiplier` (PRAIRIE 0.05x, GARRIGUE 0.15x, BOIS_JEUNE 0.55x, FORET_MATURE 0.95x, FORET_VIEILLE 1.0x)
- Érosion par foot traffic : chaque agent qui passe sur une cellule incrémente `ravine_depth` proportionnellement à sa vitesse et au niveau d'eau local. Ravine > 0.3 réduit food, > 0.6 réduit wood.

- max_ravine_depth = 1.0 → sentiers visibles formés
- 0 erreurs
