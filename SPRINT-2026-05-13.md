## Suite — P1 — Brancher God Avatar dans dashboard.py ✅

**Résultat :** ✅ **P1 PASSÉ.** 11 / 11 checks.

### Approche

Extension non-invasive de `engine/dashboard.py` : ajout d'une fonction
`start_god_server(sim, ctl, ...)` en **append** (pas de rewrite de
`start_server`). Cette fonction :
1. Instancie `GodObserver()` et `GodInterventionLog()`.
2. Appelle `register_god_endpoints(_Handler, god, log)` AVANT le démarrage
   du serveur — les routes `/api/god/*` sont prioritaires, fall-through
   vers les routes dashboard existantes pour le reste.
3. Lance `start_server(...)` et retourne `(srv, god, log)` pour
   introspection externe (tests, scripts d'intervention).

### Endpoints testés

| Méthode | Route                  | Résultat |
|---------|------------------------|----------|
| GET     | `/api/god/state`       | 200 ✅ avec `pos` + `visible` |
| POST    | `/api/god/teleport`    | 2xx ✅ god.pos mis à jour       |
| POST    | `/api/god/visibility`  | 2xx ✅ god.visible = True       |
| GET     | `/api/events/recent`   | 200 ✅ (fall-through intact)    |

`GodInterventionLog` enregistre 2 entrées (`teleport`, `visibility`) au
bon ordre, accessibles via `log.recent(5)` qui retourne des dicts JSON.

## Suite — P2 — Audio endpoints + overlay ✅

**Résultat :** ✅ **P2 PASSÉ.** 9 / 9 checks.

### Approche

Encore une extension append-only dans `engine/dashboard.py` : nouvelle
fonction `start_full_observation_server(sim, ctl, ...)` qui :

1. crée (ou récupère depuis `sim`) un `SoundField` et un
   `KnowledgeRegistry`, et les attache à `sim` (`sim.sound_field`,
   `sim.knowledge_registry`) pour que la boucle de tick puisse y émettre ;
2. monkey-patche le `_Handler` avec **god** puis **audio** dans cet
   ordre (audio s'enroule autour de god) ;
3. démarre le serveur HTTP une seule fois via `start_server`.

Retour : `(srv, god, log, sound_field, knowledge_registry)`.

Pour le HUD client : un `<script src="/static/audio_overlay.js"></script>`
a été injecté dans `engine/god_view.html` juste avant `</body>`
(append-only, pas de modification du body existant).

### Endpoints testés

| Méthode | Route                                    | Résultat |
|---------|------------------------------------------|----------|
| GET     | `/api/audio?listener_x=0&listener_y=0`   | 200 ✅ retourne 1 utterance audible |
| GET     | `/static/audio_overlay.js`               | 200 ✅ `application/javascript` (~9.9 KB) |
| GET     | `/api/god/state` (fall-through)          | 200 ✅ chaîne god + audio + dashboard préservée |

Le test seed une utterance via `sound_field.emit(...)` à (1,1,1) puis
écoute depuis (0,0). Le payload retourne `{"utterances": [...], "listener", "tick"}`.
