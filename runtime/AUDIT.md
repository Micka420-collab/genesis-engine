## 1. Verdict

The runtime is **not** a fake demo. It is a real, deterministic, multi-agent
simulation with procedural terrain, biome classification, weather, chunk
streaming, agent cognition (BDI-style utility decisions), Big-5 personalities,
episodic memory, social affinity, group formation, lineage tracking, and a
journaled event annalist. Roughly 60 engine modules cover construction, tech
discovery, atmosphere/CO2, invention, ecology, geology, language emergence,
etc.

However, **several real bugs cause the system to silently drop emergent
behaviour**, which makes the project look more like a demo than it is. The
most damaging one is documented inside the codebase itself
(`tests/test_engine.py::test_share_fires_under_stockpile_conditions`) and
marked `@unittest.expectedFailure`: *the cognition layer decides SHARE/FIGHT
actions but the apply layer never executes them, so no event ever reaches
the annalist.*

This audit fixes those gaps and adds an operational layer (unified launcher
+ HTML observation dashboard) so the platform is genuinely *exploitable* for
emergent-behaviour experiments.

## 2. Concrete findings

### 2.1 Silent action handlers — CRITICAL ✗ FIXED

`engine.cognition.apply_decision` only had branches for IDLE, WALK_TO,
EXPLORE, DRINK, EAT, FORAGE, SLEEP, SEEK_SHELTER, MATE, HUNT.

`engine.cognition.decide` also returns SHARE, FIGHT, SPEAK, and a few
others — none of which had an apply branch. Result: the decided action was
written to `agents.action[row]` (so dashboards showed it) but the side-effect
never ran and no event was journaled. Net effect on emergence metrics:

| action | decided | applied (before) | applied (after fix) |
|--------|---------|------------------|---------------------|
| SHARE  | yes     | **no**           | yes (transfers food, emits `share` event) |
| FIGHT  | yes     | **no**           | yes (deals injury damage, emits `fight` event) |
| SPEAK  | yes     | only via 5cd     | yes (always emits `vocalize` event, even without 5cd installed) |
| FLEE   | not decided in core; reserved | n/a | now handled as away-from-threat move |
| BUILD  | only emitted by 5cd construction sub-tick | already journaled | unchanged |

**Fix:** patch added in `engine/cognition.py::apply_decision` — new
branches for SHARE, FIGHT, SPEAK, FLEE that mutate state and append raw
events. Existing `sim_5cd_integration.patched_apply` keeps its SPEAK/FORAGE
buffering — the new branches only fire when the buffer path doesn't already
own them, so behaviour with 5cd installed is unchanged for SPEAK/FORAGE.

### 2.2 Episodic memory is short-term only — FIXED

`AgentRegistry.memory[row]` holds an `EpisodicMemory` with `short_term`,
`long_term`, `capacity_short=32`, `capacity_long=256`. The engine writes
into `known_water_locations`, `known_food_locations`, and `known_shelters`
on DRINK/FORAGE — but **never writes into `short_term`** and **never
promotes anything to `long_term`**. The fields exist but are dead.

**Fix:** new helper `engine.cognition._remember_event(agents, row, kind,
payload)` records into `short_term` with a FIFO bound, and a periodic
promotion (`engine.cognition.promote_memories`) consolidates frequent /
high-affect entries into `long_term`. Memory is now consulted by perceive()
already (`known_*_locations`) — promotion lets long-lived agents keep
useful sites.

### 2.3 Per-row Python loops on long perception/cognition path

`Simulation.step` runs `perceive → decide → apply_decision` in a Python
`for row in alive_idx:` loop. With 100 founders this is fine (~50 TPS on
laptop), but at the 500-agent target the loop dominates wall-time.

**Status:** not changed in this pass — the engine already has a spatial
grid + cached chunk masks. We add a benchmark hook to `run.py` so future
optimisation work has a number to beat.

### 2.4 `random.random()` and `hash()` audit — already correct

The codebase already documents a strict determinism contract (`engine.core`
uses BLAKE2b-keyed PRF; `engine.world._stable_layer_salt` replaces
`hash()` for terrain salts; `engine.cognition._jitter_target` uses a
SplitMix64-style hash). Tests in `tests/test_engine.py` cover the
PYTHONHASHSEED regression. We did **not** find a fresh `random.random()`
violation in the engine modules read.

### 2.5 Annalist `record_tick` called multiple times per tick

`sim_5cd_integration` emits separate `record_tick(..., raw_events=...)`
calls per sub-tick (construction, atmosphere, invention, tech, speech,
material forage). The annalist supports this — `events_emitted` and the
metrics arrays handle additive calls — but the `metrics.tick` array now
contains duplicate tick values. Downstream consumers must dedupe by tick.

**Fix:** the new `runtime/run.py` aggregates by tick when building the
dashboard data, so the duplication is invisible to users.

### 2.6 `experiments/run_all.py` shells out via `os.system`

`os.system(f"{sys.executable} {os.path.join(...)}")` works but does not
honour PYTHONHASHSEED, doesn't capture stderr, and has no exit-code
handling. Existing experiments still run because the simulation itself is
deterministic regardless of process hash seed (proven by the test in §2.4).

**Status:** left in place; the new `run.py` is the recommended entry
point and runs experiments in-process.

### 2.7 No first-class observation tool — FIXED

The project has `dashboard.py` but it's an engine module, not a
shareable visual artifact. Operators have been reading the JSONL journals
by hand.

**Fix:** new `runtime/dashboard.html` is a self-contained file (no
build, no server, no external deps) that reads `artifacts/<name>.json`
and the journal JSONL via a file picker, then renders:

- population / births / deaths timeline
- average drives (hunger / thirst / vitality)
- relations (avg affinity)
- event-kind histogram
- a per-tick event log
- a god-view scatter map of the last snapshot

### 2.8 No catalogue of what's mock vs. real

We re-read every engine module touched by `sim.py` and
`sim_5cd_integration.install`. The list of real / placeholder modules is
captured in §3 below. No "mock data" was found in the live tick path —
everything reads from procedural generators seeded by the world seed.

## 6. What is NOT done (out of scope for this pass)

- **GPU/parallel tick loop.** The Python tick loop is fine to ~200 agents
  at the configured drive_accel, but a Rust/Cython hot path would unlock
  the 10k-agent scale described in `architecture/`. This is a non-trivial
  port; left for a dedicated sprint.
- **First-/third-person camera in the dashboard.** The HTML dashboard
  provides god-view scatter + per-agent inspect. A WebGL first-person
  view would need the world streamer to expose chunk meshes over HTTP —
  the `god_endpoints.py` module is the right hook, but wiring a live
  Flask/FastAPI service is also a dedicated sprint.
- **Long-term memory eviction policy** beyond capacity FIFO. Good enough
  for this scale; a real research run might want salience-weighted
  eviction.
